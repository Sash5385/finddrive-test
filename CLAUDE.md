# FindDrive — CLAUDE.md

## Що це

FindDrive — маркетплейс пошуку інструкторів з водіння за геолокацією.
Учні знаходять інструктора поряд, інструктор працює з фіксованої точки — локація є головною цінністю.

Окремий проект, але планується інтеграція з **ID4Drive** (спільна підписка дешевша).

## Репозиторій

- GitHub: `sash5385/finddrive` → GitHub Pages
- Домен: `finddrive.in.ua` (apex-домен, A-записи на GitHub Pages)
- Гілка для деплою: `main`

## Поточний стек

Зараз — один статичний файл `index.html` (vanilla HTML/CSS, без JS-фреймворків).

Зовнішні залежності (CDN):
- Шрифт: `Inter` (Google Fonts)
- Іконки: `@tabler/icons-webfont` (Tabler Icons, клас `ti ti-*`)

## Структура лендінгу (index.html)

1. **Навбар** — логотип FindDrive, посилання (Інструктори / Як це працює / Відгуки), кнопки "Увійти" і "Стати інструктором"
2. **Hero** — blob-фон, пошук по локації, кнопка геолокації "Моє місце", популярні райони (Шевченківський, Оболонь, Позняки, Відрадний)
3. **Картки інструкторів** — сітка 3 колонки, аватар-ініціали, рейтинг, відстань, теги, ціна/год, кнопка "Записатись"
4. **Як це працює** — 4 кроки в кольорових картках (зелена, рожева, синя, оливкова)
5. **Карта** — placeholder із SVG-сіткою і піна-мітками інструкторів
6. **Footer** — логотип, посилання, копірайт

## UI-правила (панелі та вікна)

### Стиль усіх вікон і модалок — ОБОВ'ЯЗКОВО

**Будь-яке вікно, модалка, панель, drawer, popup** — має відповідати дизайн-системі сайту:

- Фон: `var(--white)` або `var(--green-light)` — **ніколи не чисто білий без акцентів**
- Шапка (header) вікна: фон `var(--green)` або `var(--green-light)`, текст відповідно
- Обводка: **`border: 1px solid #000`** (тонка чорна лінія по периметру)
- `border-radius`: 18px для модалок, 0 або 18px для бокових панелей
- Кнопки, інпути, бейджі — тільки з палітри дизайн-системи (`--green`, `--green-light`, `--text`, тощо)
- Overlay (підложка): `rgba(13,42,26,.5)` з `backdrop-filter:blur(4px)`

### Панель профілю інструктора (`instr-profile-panel`)

- Ширина: **`min(96vw, 480px)`** (mobile) → **`720px`** (≥900px) → **`960px`** (≥1280px)
- Ширша за звичайний кабінет — бо показує повний профіль + вибір дати/часу для запису
- Всі кнопки і текст у панелі — симетричні і логічно розташовані

### Що показувати клієнту в профілі інструктора

- **Показувати**: фото, ціна, стаж, район, теги, опис, точки зустрічі, ТСЦ, відгуки, телефон, **календар + відкриті слоти**
- **НЕ показувати**: графік роботи інструктора (`days`, `time_from`–`time_to`) — клієнту достатньо бачити доступні дати та слоти

## Бізнес-модель

- **Монетизація**: підписка для інструкторів (після набору критичної маси)
- **Цінність**: локація — учні доїжджають до інструктора самі
- **Інтеграція з ID4Drive**: спільна підписка дешевша ніж окремо

## Версіонування та деплой

### Правила

1. **Кожна зміна супроводжується записом у `CHANGELOG.md`** перед комітом.
2. Використовується **дата-версіонування** (`YY.MM.NN`):
   - `YY` — рік (24, 25…)
   - `MM` — місяць (06, 07…)
   - `NN` — порядковий номер релізу за цей день (01, 02, 03…)
   - Приклад: перший реліз 24 червня → `24.06.01`, другий → `24.06.02`
3. **Версія оновлюється в `index.html`** — бейдж `<span class="logo-version">v24.06.04</span>` у навбарі.
4. **АВТОДЕПЛОЙ — обов'язково після кожного коміту**: одразу мерджити в `main` і пушити. GitHub Pages оновиться автоматично.

### Воркфлоу кожного оновлення (АВТОДЕПЛОЙ)

```
1. Внести зміни в код
2. Оновити CHANGELOG.md — додати новий запис зверху з наступним номером
3. Оновити версію в index.html (logo-version у навбарі)
4. git add . && git commit -m "..."
5. git push -u origin claude/<branch>
6. # АВТОДЕПЛОЙ — завжди одразу після commit:
   git checkout main
   git merge --ff-only claude/<branch> || git checkout --theirs index.html CHANGELOG.md && git add . && git commit -m "merge"
   git push origin main
   git checkout claude/<branch>
```

### Формат запису в CHANGELOG.md

```markdown
## [24.06.05] — 2026-06-24

### Added
- Опис нового

### Changed
- Що змінилось

### Fixed
- Що виправлено
```
